# 项目经济账本（Flask）

一个最小可用的内部项目收支系统：支持登录、项目、收入/支出流水、凭证上传、项目概览与报表。

## 功能概览

- **登录与用户**：账号登录、修改密码、管理员用户管理（新增/编辑/重置密码）
- **项目**：创建项目、项目成员、项目详情页查看资金与进展
- **流水**：
  - 项目成员可发起 **新增流水申请**（先进入“待审核流水”，审核通过后生效并进入统计）
  - 项目成员可对已生效流水发起 **修改申请** / **删除申请**
  - 《审核通过》规则：**项目全体成员同意** 或 **管理员同意**
  - 管理员拥有绝对权力：可随时执行生效/修改/删除
- **凭证**：支持中文文件名展示；磁盘存储按项目分目录 `uploads/projects/<项目ID>/...`
- **项目日志**：系统自动记录关键操作（创建/编辑项目、流水新增/修改流程、分红、删除、结束日期变更等）
- **权限（摘要）**：
  - **管理员**：可执行敏感操作（如执行修改/删除/变更等）
  - **普通用户**：仅可查看自己所在项目；可创建项目；项目详情按成员关系校验

## 目录结构（简）

- `run.py`：本地开发启动入口
- `ledger_app/`：Flask 应用代码
- `templates/`：Jinja2 模板
- `instance/ledger.db` 或 `ledger.db`：SQLite 数据库（首次启动自动创建）
- `uploads/`：凭证与附件存储目录（可配置）
- `scripts/`：运维脚本（systemd 启停等）

## 启动

在 `e:\项目管理` 目录执行：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt
python run.py
```

浏览器打开 `http://127.0.0.1:5000`。

## 配置项（环境变量）

- `SECRET_KEY`：Flask 密钥（生产环境务必设置）
- `DATABASE_URL`：数据库连接串，默认 `sqlite:///ledger.db`
- `UPLOAD_FOLDER`：上传目录，默认项目根目录下的 `uploads/`
- `URL_PREFIX`：如需部署在子路径（例如 `/PM`），设置该前缀

## 初始管理员账号

- 用户名：`admin`
- 密码：`admin123!`

首次启动会自动创建 `ledger.db`（SQLite）并写入初始管理员。

## 金额单位

界面输入与展示均为 **元**。系统内部会转换为“分”保存（避免小数误差）。

## 凭证上传与访问

- **上传位置**：默认保存到 `uploads/` 下，并按项目分目录组织：`uploads/projects/<项目ID>/...`
- **访问**：页面上通过 `/uploads/<path>` 访问（已做成员权限校验）
- **提示**：若部署到生产环境，建议将 `UPLOAD_FOLDER` 配置到持久化磁盘，并做好备份策略

## 用户管理

- 管理员可在导航栏进入 **用户** 页面：`/users`
- **新增用户**：默认初始密码为 `123456!`（也可手动填写初始密码）
- **修改密码**：右上角 **账户 → 修改密码**（需要输入当前密码）
- **重置密码**：在用户列表/编辑页点击 **重置密码**，先输入管理员密码确认，成功后弹窗提示新密码为 `123456!`

## 流水修改（纠错）

- 项目成员可发起“修改申请”，成员/管理员可点击 **同意修改**；满足《审核通过》后点击 **执行修改** 生效
- 项目成员可发起“删除申请”，满足《审核通过》后点击 **执行删除**（会将流水作废，便于追溯）
- 项目成员新增流水默认进入 **待审核**，满足《审核通过》后点击 **执行生效**

## 项目日志

项目详情页包含「项目日志」模块，会自动记录关键操作，便于事后追溯。

## 运维（systemd）

如使用 systemd 服务 `/etc/systemd/system/private-pm.service`，项目内提供：

- `scripts/stop_pm_server.sh`
- `scripts/start_pm_server.sh`
